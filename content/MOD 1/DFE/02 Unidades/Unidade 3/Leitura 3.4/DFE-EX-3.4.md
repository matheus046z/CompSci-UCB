### 1.

> [!question] Exercício 1
> De acordo com o modelo de acessibilidade em governo eletrônico (eMAG), alinhado às diretrizes da WCAG, a acessibilidade em páginas web deve ser garantida por meio de requisitos técnicos que favoreçam o acesso equitativo à informação.
>
> Assinale a alternativa que apresenta um requisito corretamente estabelecido pelo eMAG:
>
> **A.**  
> Uso exclusivo de elementos `<div>` para estruturar a página.
>
> **B.**  
> Estruturação do conteúdo com marcação semântica adequada em HTML.
>
> **C.**  
> Priorização apenas da aparência visual da interface.
>
> **D.**  
> Uso de imagens sem descrição textual.
>
> **E.**  
> Navegação dependente exclusivamente do mouse.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: B.**
>>
>> **A.** Errada. O uso exclusivo de `<div>` não garante acessibilidade e ainda prejudica a semântica da página.
>>
>> **B.** Certa. A marcação semântica em HTML ajuda tecnologias assistivas a interpretar corretamente a estrutura e o conteúdo.
>>
>> **C.** Errada. A acessibilidade não se limita ao visual; ela envolve navegação, compreensão e interação.
>>
>> **D.** Errada. Imagens informativas devem ter descrição textual adequada.
>>
>> **E.** Errada. A navegação não pode depender só do mouse, pois muitos usuários utilizam teclado ou tecnologias assistivas.
>>
>>A estruturação do conteúdo com marcação semântica adequada em HTML está correta, pois o eMAG estabelece que a acessibilidade deve ser garantida por meio do uso adequado de marcação semântica em HTML, permitindo que tecnologias assistivas interpretem corretamente a estrutura e o conteúdo da página.  
>>
>>O uso exclusivo de `<div>` está incorreto, pois não transmite significado semântico e dificulta a interpretação do conteúdo por tecnologias assistivas, contrariando o eMAG.
>>
>>Priorização da aparência da interface está incorreta, pois a acessibilidade não se limita ao visual, devendo considerar interação, navegação e estrutura do código.
>>
>>Uso de imagens sem descrição textual está incorreto porque imagens sem descrição textual criam barreiras para usuários com deficiência visual, violando as recomendações do eMAG.
>>
>>Navegação dependente exclusivamente do _mouse_ está incorreta porque o eMAG exige que a navegação seja possível também por teclado, e não apenas pelo _mouse_.
### 2.

> [!question] Exercício 2
> De acordo com o eMAG, fundamentado nas Diretrizes de Acessibilidade para Conteúdo Web (WCAG 2.1), todo conteúdo não textual presente em uma página web deve ter uma alternativa textual equivalente, de modo a garantir que a informação seja percebida por usuários que utilizam tecnologias assistivas, como leitores de tela.
>
> Considerando esse princípio, assinale a alternativa que atende corretamente às recomendações do eMAG para o tratamento de imagens em aplicações web.
>
> **A.**  
> Utilizar imagens sem atributo alt.
>
> **B.**  
> Usar textos alternativos genéricos.
>
> **C.**  
> Fornecer textos alternativos que descrevam o conteúdo ou a função da imagem.
>
> **D.**  
> Substituir imagens por estilo CSS.
>
> **E.**  
> Inserir descrição apenas no título da página.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: C.**
>>
>> **A.** Errada. Imagens sem `alt` dificultam ou impedem a compreensão por leitores de tela.
>>
>> **B.** Errada. Textos genéricos, como “imagem” ou “figura”, não transmitem a informação necessária.
>>
>> **C.** Certa. O texto alternativo deve descrever o conteúdo ou a função da imagem de forma útil ao usuário.
>>
>> **D.** Errada. CSS não substitui a necessidade de uma alternativa textual acessível.
>>
>> **E.** Errada. O título da página não substitui a descrição individual de cada imagem.
>>
>>Fornecer textos alternativos que descrevam o conteúdo ou a função da imagem está correto, pois o eMAG recomenda que o atributo "alt" descreva o conteúdo ou a função da imagem, assegurando equivalência informacional.
>>
>>Utilizar imagens sem atributo _alt_ está incorreto, pois a ausência do atributo `alt` impede que leitores de tela transmitam o significado da imagem ao usuário.
>>
>>Usar textos alternativos genéricos está incorreto porque textos alternativos genéricos não garantem a compreensão do conteúdo ou da função da imagem.
>>
>>Substituir imagens por estilos CSS está incorreto, uma vez que o uso de CSS não substitui a necessidade de alternativas textuais para conteúdos não textuais.
>>
>>Inserir descrição apenas no título da página está incorreto, pois o título da página não fornece descrição específica para cada imagem presente no conteúdo.
### 3.

> [!question] Exercício 3
> O eMAG estabelece recomendações para que aplicações web sejam acessíveis a diferentes perfis de usuários. A avaliação de conformidade não deve ser encarada como uma etapa única, mas como um processo contínuo de verificação e melhoria, envolvendo diferentes dimensões da experiência digital.
>
> Nesse sentido, considera-se correta a avaliação de conformidade que:
>
> **A.**  
> se limita a verificar apenas a página inicial.
>
> **B.**  
> analisa somente a presença de cores contrastantes.
>
> **C.**  
> avalia somente formulários e campos de entrada.
>
> **D.**  
> inclui testes automáticos, mas também inspeções manuais e uso de tecnologias assistivas.
>
> **E.**  
> foca apenas a responsividade em dispositivos móveis.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: D.**
>>
>> **A.** Errada. Avaliar só a página inicial é insuficiente para verificar a acessibilidade de todo o sistema.
>>
>> **B.** Errada. Contraste é importante, mas não é o único aspecto da acessibilidade.
>>
>> **C.** Errada. Formulários são relevantes, porém a avaliação deve abranger todo o ambiente digital.
>>
>> **D.** Certa. Uma avaliação adequada combina ferramentas automáticas com análise humana e testes com tecnologias assistivas.
>>
>> **E.** Errada. Responsividade ajuda, mas não substitui uma avaliação completa de acessibilidade.
>>
>>A alternativa que diz que inclui testes automáticos, mas também inspeções manuais e uso de tecnologias assistivas está correta, pois a avaliação adequada combina testes automatizados com inspeções manuais e práticas com tecnologias assistivas.
>>
>>"Limita-se a verificar apenas a página inicial" está incorreto, pois a acessibilidade não pode ser garantida apenas pela análise da página inicial.
>>
>>"Analisa somente a presença de cores contrastantes" está incorreta porque o contraste é relevante, mas não o suficiente para assegurar conformidade.
>>
>>"Avalia somente formulários e campos de entrada" está incorreto, pois formulários são importantes, mas representam apenas parte da aplicação.
>>
>>"Foca apenas a responsividade em dispositivos móveis está incorreta" porque responsividade é um critério, mas não abrange todos os aspectos de acessibilidade.
### 4.

> [!question] Exercício 4
> O eMAG orienta que a acessibilidade em sistemas digitais deve ser compreendida como um requisito essencial para garantir a inclusão de todos os cidadãos. Avaliar o grau de acessibilidade não significa apenas verificar aspectos técnicos isolados, mas também considerar como diferentes elementos do sistema contribuem para a autonomia e a experiência do usuário. Nesse processo, é importante reconhecer que tanto a estrutura do sistema quanto a forma de interação influenciam diretamente na efetividade da acessibilidade.
>
> Considere as afirmativas a seguir sobre a avaliação do grau de acessibilidade segundo o eMAG:
>
> 1. **A acessibilidade deve ser tratada como um atributo transversal do sistema.**  
> 2. **A conformidade pode ser avaliada apenas por ferramentas automáticas.**  
> 3. **Inconsistências na navegação comprometem a autonomia do usuário.**  
> 4. **Páginas acessíveis isoladamente garantem a acessibilidade do sistema inteiro.**
>
> É correto o que se afirma em:
>
> **A.**  
> I, apenas.
>
> **B.**  
> II, apenas.
>
> **C.**  
> I e III, apenas.
>
> **D.**  
> II e IV, apenas.
>
> **E.**  
> I, II e IV, apenas.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: C.**
>>
>> **A.** Errada. A afirmativa I está correta, mas a III também está.
>>
>> **B.** Errada. A afirmativa II está incorreta, pois ferramentas automáticas sozinhas não bastam.
>>
>> **C.** Certa. A acessibilidade deve ser tratada de forma transversal, e problemas de navegação realmente afetam a autonomia do usuário.
>>
>> **D.** Errada. Tanto a II quanto a IV estão incorretas.
>>
>> **E.** Errada. A II e a IV são falsas, então essa combinação não procede.
>>
>>"A acessibilidade deve ser tratada como um atributo transversal do sistema" está correta porque o eMAG trata acessibilidade como atributo transversal, presente em todas as etapas do sistema.
>>
>>"A conformidade pode ser avaliada apenas por ferramentas automáticas" está incorreta, pois ferramentas automáticas ajudam, mas não são suficientes para avaliação completa.
>>
>>"Inconsistências na navegação comprometem a autonomia do usuário" está correta, uma vez que >>inconsistências na navegação afetam diretamente a autonomia e a usabilidade.
>>
>>"Páginas acessíveis isoladamente garantem a acessibilidade do sistema inteiro" está incorreta, pois a acessibilidade deve ser sistêmica; páginas isoladas não garantem conformidade integral.

### 5.

> [!question] Exercício 5
> Na avaliação de acessibilidade de páginas web, conforme o eMAG e as recomendações da WCAG 2.1, é fundamental observar se os elementos da interface estão implementados de forma a garantir que usuários com diferentes perfis — incluindo aqueles que utilizam leitores de tela ou navegação por teclado — consigam interagir com o sistema. O uso de marcações semânticas adequadas e práticas de desenvolvimento acessível é parte essencial desse processo.
>
> Durante a verificação do eMAG em uma página web, identificou-se o seguinte trecho de código:
>
> `<div onclick="enviar()">Enviar</div>`
>
> De acordo com as recomendações do eMAG e da WCAG, a correção mais adequada é:
>
> **A.**  
> manter o código, pois funciona visualmente.
>
> **B.**  
> substituir por `<span>` com evento de clique.
>
> **C.**  
> utilizar `<button type="submit">Enviar</button>`.
>
> **D.**  
> inserir apenas estilos CSS para melhorar a aparência.
>
> **E.**  
> remover a funcionalidade do botão.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: C.**
>>
>> **A.** Errada. Funcionar visualmente não significa ser acessível para teclado e tecnologias assistivas.
>>
>> **B.** Errada. `<span>` continua sem semântica apropriada para ação de botão.
>>
>> **C.** Certa. O elemento `<button>` é semanticamente correto, acessível por teclado e melhor interpretado por leitores de tela.
>>
>> **D.** Errada. CSS melhora aparência, mas não corrige problema semântico e de acessibilidade.
>>
>> **E.** Errada. O objetivo é tornar a funcionalidade acessível, não removê-la.
>>
>>Utilizar `<button type="submit">Enviar</button>` está correto, pois `<button>` é semântico, acessível e operável por teclado, atendendo às recomendações do eMAG e da WCAG.
>>
>>Manter o código, pois funciona visualmente, é incorreta porque funcionamento visual não garante acessibilidade.
>>
>>Substituir por `<span> `com evento de clique está incorreto porque a tag `<span>` também é elemento genérico, sem semântica adequada.
>>
>>Inserir apenas estilos CSS para melhorar a aparência está incorreto, uma vez que CSS melhora a aparência, mas não resolve problemas semânticos e de interação.
>>
>>Remover a funcionalidade do botão está incorreto, pois a funcionalidade deve ser mantida, mas de forma acessível.
