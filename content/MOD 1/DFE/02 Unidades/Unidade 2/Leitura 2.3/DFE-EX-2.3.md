### 1
> [!question] Exercício 1
> Podemos dividir uma ação feita com o JavaScript em dois elementos básicos. Quais são?
>
> **A.**  
> Variável e constante.
>
> **B.**  
> Função e evento.
>
> **C.**  
> Função e variável.
>
> **D.**  
> IF e else.
>
> **E.**  
> Evento e execução.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: B.**
>>
>> **A.** Errada. Variável e constante são formas de armazenar dados, mas não representam os dois elementos básicos de uma ação em JavaScript.
>>
>> **B.** Certa. Em JavaScript, uma ação normalmente envolve um **evento**, que dispara algo, e uma **função**, que executa o comportamento desejado.
>>
>> **C.** Errada. A função participa da ação, mas a variável não é um elemento básico que dispara ou estrutura a ação.
>>
>> **D.** Errada. `if` e `else` são estruturas condicionais, usadas para decisão lógica, não para definir os dois elementos básicos de uma ação.
>>
>> **E.** Errada. O evento faz parte do processo, mas “execução” não é normalmente tratado como o par básico formal nessa divisão; o mais correto é **função e evento**.


### 2
> [!question] Exercício 2
> São exemplos de eventos do JavaScript:
>
> **A.**  
> OnClick, OnCharge e OnLoad.
>
> **B.**  
> OnClick, OnSubmit e OnLoad.
>
> **C.**  
> OnForward, OnSubmit e OnLoad.
>
> **D.**  
> OnClick, OnSubmit e OnRewind.
>
> **E.**  
> OnClick, OnReplay e OnLoad.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: B.**
>>
>> **A.** Errada. `OnClick` e `OnLoad` são exemplos de eventos, mas `OnCharge` não é um evento padrão do JavaScript/HTML.
>>
>> **B.** Certa. `OnClick`, `OnSubmit` e `OnLoad` são eventos clássicos usados em páginas web para clique, envio de formulário e carregamento da página.
>>
>> **C.** Errada. `OnSubmit` e `OnLoad` existem, mas `OnForward` não é um evento padrão do JavaScript/HTML.
>>
>> **D.** Errada. `OnClick` e `OnSubmit` existem, porém `OnRewind` não é um evento padrão.
>>
>> **E.** Errada. `OnClick` e `OnLoad` existem, mas `OnReplay` não é um evento padrão do JavaScript/HTML.

### 3
> [!question] Exercício 3
> São métodos de JavaScript bastante utilizados:
>
> **A.**  
> OnClick, OnSubmit e OnLoad.
>
> **B.**  
> Alarm(), Confirm() e Play().
>
> **C.**  
> Alert(), Confirm() e Prompt().
>
> **D.**  
> if, elseif e else.
>
> **E.**  
> DOM, Canvas e Object.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: C.**
>>
>> **A.** Errada. `OnClick`, `OnSubmit` e `OnLoad` são eventos, não métodos.
>>
>> **B.** Errada. `Confirm()` existe, mas `Alarm()` e `Play()` não formam o conjunto clássico de métodos básicos mais usados nesse contexto.
>>
>> **C.** Certa. `Alert()`, `Confirm()` e `Prompt()` são métodos bastante conhecidos do JavaScript para interação com o usuário por meio de caixas de diálogo.
>>
>> **D.** Errada. `if`, `elseif` e `else` são estruturas condicionais, não métodos.
>>
>> **E.** Errada. `DOM`, `Canvas` e `Object` são conceitos ou objetos relacionados ao JavaScript, mas não são esse conjunto de métodos.


### 4
> [!question] Exercício 4
> **Qual a sintaxe correta de um código JavaScript?**
>
> **A.**  
> `<script type="text/css"> conteúdo do Script; </script>`
>
> **B.**  
> `<script type="text/javascript"> conteúdo do Script; </script>`
>
> **C.**  
> `<Javascript> conteúdo do Script; </Javascript>`
>
> **D.**  
> `<Java> conteúdo do Script; </Java>`
>
> **E.**  
> `<script type="java"> conteúdo do Script; </script>`
>
>> [!question]- Resposta
>>
>> **Alternativa correta: B.**
>>
>> **A.** Errada. `text/css` é o tipo usado para CSS, não para JavaScript.
>>
>> **B.** Certa. A tag `<script>` é a forma correta de inserir JavaScript em HTML, e `type="text/javascript"` representa a sintaxe clássica correta.
>>
>> **C.** Errada. A tag `<Javascript>` não existe em HTML.
>>
>> **D.** Errada. A tag `<Java>` também não existe para inserir código JavaScript em HTML.
>>
>> **E.** Errada. `type="java"` está incorreto, porque Java e JavaScript são linguagens diferentes.

### 5
> [!question] Exercício 5
> Qual a função de uma VARIÁVEL no JavaScript?
>
> **A.**  
> Fazer um teste condicional.
>
> **B.**  
> Ter um valor fixo que não pode ser alterado.
>
> **C.**  
> Ter um valor que varia ou que pode ser inserido pelo usuário.
>
> **D.**  
> Ter um valor nulo.
>
> **E.**  
> Não existem variáveis no JavaScript.
>
>> [!question]- Resposta
>>
>> **Alternativa correta: C.**
>>
>> **A.** Errada. Fazer teste condicional é função de estruturas como `if`, e não de uma variável.
>>
>> **B.** Errada. Um valor fixo que não pode ser alterado se relaciona mais à ideia de constante, não de variável.
>>
>> **C.** Certa. A variável serve para armazenar dados que podem mudar durante a execução do programa ou receber valores informados pelo usuário.
>>
>> **D.** Errada. Uma variável pode até receber valor nulo, mas essa não é sua função principal.
>>
>> **E.** Errada. JavaScript possui variáveis, como as declaradas com `let`, `var` e `const`.
