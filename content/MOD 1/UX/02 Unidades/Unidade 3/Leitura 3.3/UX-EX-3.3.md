Para manter a sua simplicidade, ela não é tão completa quanto as outras ferramentas. ### 2
> [!question] Exercício 2
> Com o avançar do projeto de desenvolvimento, as técnicas utilizadas para a construção da interface vão mudando, a fim de atender às necessidades emergentes. Dessa forma, a criação de mockups e wireframes se torna uma importante fase no processo de estruturação da interface e avaliação do projeto. Sobre wireframes e mockups, considere as afirmativas a seguir e assinale alternativa correta:
>
> I – Os mockups são um meio termo entre representações de baixa fidelidade e protótipos de alta fidelidade, que são mais complexos de serem construídos.
>
> II – Os wireframes têm uma representação mais realista, com um maior grau de fidelidade da interface do produto final.
>
> III – Para a criação dos mockups, ainda não existe uma preocupação com a estética do sistema. Busca-se avaliar como as informações serão apresentadas.
>
> **A.**  
> Apenas a I está correta.
>
> **B.**  
> Apenas a II está correta.
>
> **C.**  
> Apenas a III está correta.
>
> **D.**  
> Apenas a I e a II estão corretas.
>
> **E.**  
> Apenas a I e a III estão corretas.
>> [!question]- Resposta
>>
>> **Resposta correta: A. Apenas a I está correta.**
>>
>> **Análise das afirmativas:**
>>
>> **I. Certa.**  
>> Os **mockups** realmente funcionam como uma etapa intermediária entre representações mais simples e protótipos de alta fidelidade. Eles apresentam mais detalhes visuais do que wireframes, mas ainda são menos complexos do que um protótipo altamente interativo.
>>
>> **II. Errada.**  
>> Os **wireframes** não têm representação mais realista. Em geral, eles são estruturas mais simples da interface, focadas em organização, disposição dos elementos e fluxo de informação, com **baixa fidelidade visual**.
>>
>> **III. Errada.**  
>> Nos **mockups**, já existe, sim, uma preocupação maior com a **estética da interface**, como cores, tipografia, ícones e aparência visual. A ideia de focar apenas na apresentação das informações, sem preocupação estética, se aproxima mais dos wireframes.
>>
>> São os _wireframes_ que têm uma baixa fidelidade. Eles são utilizados para comunicar a ideia apresentada pela equipe e compreender como propor uma solução para o problema enfrentado. Com os _wireframes_ são apresentados apenas elementos que comporão o sistema, mas sem forma, cor e fonte definidas. Todavia, os _mockups_ são representações de maior nível de fidelidade, em comparação aos _wireframes_. Eles têm representações mais realistas e já são compostos por decisões de design, como cor, _template_, etc. Dessa forma, quando falamos de _wireframe_, ainda não há preocupação com design estético, mas quando falamos de _mockups_, sim.

### 3
> [!question] Exercício 3
> Os protótipos de alta fidelidade possibilitam a avaliação da estrutura da informação, interação e design. A partir de uma validação do projeto, a equipe de desenvolvimento dará vida ao sistema desenhado. Dessa forma, ferramentas podem ser utilizadas para guiar o desenvolvedor na realização dessa tarefa. Sobre o processo de implementação da interface, considere as afirmativas a seguir e assinale a alternativa correta:
>
> I – Uma Interface de Programação de Aplicativos (API) fornece padrões que devem ser seguidos para a construção da aplicação.
>
> II – Os Plugins são pequenos programas instalados em programas maiores, que apresentam uma extensão em suas funcionalidades.
>
> III – Os frameworks facilitam a comunicação do desenvolvedor com linguagens de programação. Podem existir frameworks para desenvolvimento front-end e back-end.
>
> **A.**  
> Apenas a I está correta.
>
> **B.**  
> Apenas a II está correta.
>
> **C.**  
> Apenas a III está correta.
>
> **D.**  
> Apenas a II e a III estão corretas.
>
> **E.**  
> I, II e III estão corretas.
>> [!question]- Resposta
>>
>> **Resposta correta: E. I, II e III estão corretas.**
>>
>> **Análise das afirmativas:**
>>
>> **I. Certa.**  
>> Uma **API** define formas padronizadas de comunicação e uso de recursos entre sistemas, bibliotecas ou componentes, funcionando como um conjunto de regras e padrões para a construção da aplicação.
>>
>> **II. Certa.**  
>> **Plugins** são extensões adicionadas a programas maiores para ampliar ou complementar suas funcionalidades.
>>
>> **III. Certa.**  
>> **Frameworks** oferecem estrutura, organização e recursos prontos para o desenvolvimento, facilitando o trabalho do programador. Além disso, existem frameworks tanto para **front-end** quanto para **back-end**.
>>
>> As APIs disponibilizam bibliotecas e recursos que devem ser implementados para possibilitar uma melhor construção do sistema. Elas são construídas para facilitar o processo de desenvolvimento, guiando o designer na realização de sua atividade. Elas podem se apresentar de diferentes formas, as mais conhecidas são _plugins_ e _framework_. Os _plugins_ são sistemas que podem ser instalados em sistemas maiores, adicionando uma funcionalidade a mais ao sistema que os recebe. Um exemplo comum é o _plugin_ do navegador. Os _frameworks_ são ferramentas voltadas para o desenvolvimento de sistemas computacionais. Diversos _frameworks_ podem ser utilizados em um projeto. Existem os _frameworks_ de _fron-end_ para a criação de interface do sistema, quando de _back-end_.

> [!question] Exercício 4
> A implementação de interface é um processo a ser realizado quando a estrutura da interface já está definida, para que não haja desperdício de tempo e recurso. Para o desenvolvimento de interfaces, vários frameworks minimizam o esforço do desenvolvedor, apresentando diversos recursos que facilitam a implementação. Sobre os principais frameworks disponíveis, avalie as alternativas a seguir e assinale a correta:
>
> I – O Pure utiliza as linguagens HTML e CSS para o projeto de interface web, possibilitando a criação de interfaces responsivas.
>
> II – O Semantic-UI é um framework livre e de código aberto, que utiliza o padrão Material Design da Google e tem como diferencial uma linguagem “compatível com humanos”.
>
> III – O Foundation é um framework para qualquer dispositivo, sendo completamente personalizável, possibilitando a inclusão e a exclusão de elementos.
>
> **A.**  
> Apenas a I está correta.
>
> **B.**  
> Apenas a II está correta.
>
> **C.**  
> Apenas a III está correta.
>
> **D.**  
> Apenas a I e a II estão corretas.
>
> **E.**  
> Apenas a I e a III estão corretas.
>> [!question]- Resposta
>>
>> **Resposta correta: E. Apenas a I e a III estão corretas.**
>>
>> **Análise das afirmativas:**
>>
>> **I. Certa.**  
>> O **Pure** é voltado para interfaces web e trabalha com **HTML e CSS**, oferecendo recursos para construção de layouts **responsivos**.
>>
>> **II. Errada.**  
>> O **Semantic-UI** é realmente livre, de código aberto e tem como diferencial a linguagem “compatível com humanos”, mas a afirmação erra ao dizer que ele **utiliza o padrão Material Design da Google**. Essa característica está associada a outros frameworks, não ao Semantic-UI.
>>
>> **III. Certa.**  
>> O **Foundation** é um framework pensado para diferentes dispositivos e tem como característica importante ser **completamente personalizável**, permitindo adaptar, incluir e excluir elementos conforme a necessidade do projeto.
>>
>>
>> O Pure é uma ferramenta que se apresenta como simples e de fácil aprendizado. Ele é voltado para a criação de sistemas web, por isso utiliza apenas CSS e HTML. Seus sistemas são mais leves, pois ele não utiliza a linguagem JavaScript. O Semantic-UI busca apresentar TAGs de forma mais compreensível para usuário e computador, de forma a facilitar a criação da interface. Porém, ele não se baseia no padrão Material Design. O Fundation é um dos _frameworks_ mais utilizados no mundo, por ser uma ferramenta simples, intuitiva e flexível, permitindo que seus usuários desenvolvam para qualquer dispositivo, personalizando-o de acordo com a sua vontade.

> [!question] Exercício 5
> O processo de avaliação da interface tem por principal objetivo encontrar erros antes do sistema ser entregue ao cliente, diminuindo o transtorno e a ocorrência de problemas futuros. A realização da avaliação se faz importante, mesmo a equipe de desenvolvimento tendo seguido um processo de qualidade. Em IHC, existem quatro critérios que são estabelecidos como qualidade em uma aplicação. Considere as afirmativas a seguir e assinale a alternativa correta:
>
> I – A realização da avaliação automatizada necessita de um sistema funcional para a sua aplicação.
>
> II – A ferramenta THEM possibilita a avaliação automatizada do sistema computacional, considerando as heurísticas de Nielsen.
>
> III – O Wave é uma ferramenta desktop que possibilita avaliar a comunicabilidade dos sistemas computacionais.
>
> **A.**  
> Apenas a I está correta.
>
> **B.**  
> Apenas a II está correta.
>
> **C.**  
> Apenas a III está correta.
>
> **D.**  
> Apenas a I e a II estão corretas.
>
> **E.**  
> Apenas a I e a III estão corretas.
>> [!question]- Resposta
>>
>> **Resposta correta: D. Apenas a I e a II estão corretas.**
>>
>> **Análise das afirmativas:**
>>
>> **I. Certa.**  
>> A avaliação pode ser realizada desde o início do processo de design, mas, para o uso de **ferramentas automatizadas**, é necessário que exista ao menos **uma versão funcional do sistema**, já que essas ferramentas analisam a estrutura e o código da aplicação.
>>
>> **II. Certa.**  
>> A ferramenta **THEM** foi desenvolvida com o objetivo de **automatizar a avaliação de usabilidade**, permitindo ao avaliador registrar problemas encontrados com base nas **heurísticas de Nielsen**.
>>
>> **III. Errada.**  
>> O **Wave** não é uma ferramenta desktop voltada à **comunicabilidade**. Seu foco está na **avaliação de acessibilidade** de sistemas web, tomando como referência diretrizes como as **WCAG 2.0**.
>>
>> O processo de avaliação pode ser realizado desde o início do processo de design, porém, para a utilização de ferramentas automatizadas, faz-se necessário que haja pelo menos uma versão funcional do sistema. As ferramentas disponíveis utilizam-se do código do sistema para compreender a estrutura apresentada para que haja a realização da análise. 
>> O **THEM** foi desenvolvido como uma **ferramenta colaborativa que visa a automatizar a avaliação de usabilidade do sistema**. Ele possibilita ao avaliador realizar marcações de problemas encontrados, **tomando como base as heurísticas de Nielsen**. 
>> O **Wave** se mostra como outra ferramenta que **automatiza a avaliação de sistemas web**, porém, **seu foco é a acessibilidade do sistema**, tendo como parâmetro as diretrizes de acessibilidade desenvolvidas pela W3C, conhecidas como WCAG 2.0.
