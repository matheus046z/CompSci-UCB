### 1
> [!question] Exercício 1
> Você trabalha em uma empresa que produz software. Atualmente, você está envolvido em um novo projeto de interface, tendo a atribuição de selecionar os usuários participantes da coleta de dados de uma interface de site de comércio eletrônico. Neste caso, quais são os tipos de usuários mais adequados para participar do processo?
>
> **A.**  
> Usuários finais primários.
>
> **B.**  
> Usuários finais secundários.
>
> **C.**  
> Usuários finais e demais stakeholders do projeto.
>
> **D.**  
> Funcionários do atendimento ao cliente.
>
> **E.**  
> Usuários finais primários e secundários.
>> [!question]- Resposta
>>
>> **Resposta correta: C. Usuários finais e demais stakeholders do projeto.**
>>
>> **Análise das alternativas:**
>>
>> **A. Errada.**  
>> Os **usuários finais primários** são importantes, mas não são os únicos adequados. Na coleta de dados para design de interação, também devem ser considerados outros interessados no sistema, isto é, os **stakeholders**.
>>
>> **B. Errada.**  
>> Os **usuários finais secundários** também podem contribuir, mas isoladamente não representam todos os perfis relevantes para levantar dados de uma interface.
>>
>> **C. Certa.**  
>> Em IHC e design de interação, a coleta de dados deve envolver **usuários finais** e também **pessoas interessadas no sistema (stakeholders)**, porque ambos ajudam a identificar necessidades, impactos e requisitos do projeto.
>>
>> **D. Errada.**  
>> Funcionários do atendimento ao cliente podem ser fontes úteis em alguns contextos, mas não substituem os **usuários finais** nem abrangem todos os **stakeholders** do projeto.
>>
>> **E. Errada.**  
>> Incluir **usuários finais primários e secundários** é melhor do que considerar apenas um grupo, mas ainda falta contemplar os **demais stakeholders**, que também devem participar da coleta de dados.
>>
>> Quando se está produzindo um _site_ de comércio eletrônico, é importante que os usuários participantes da coleta de dados sejam representantes de todos os _stakeholders_ envolvidos no projeto, como usuários finais, clientes, fornecedores, pessoal de atendimento ao cliente, técnicos e gestores da empresa que está solicitando o desenvolvimento da interface, pois é preciso conhecer a maneira como cada um vai realizar suas atividades com o uso do artefato.


### 2
> [!question] Exercício 2
> A empresa na qual você trabalha como designer de interface recebeu um colega novo na equipe há algum tempo. Como primeira atividade, ele foi designado para trabalhar no levantamento de requisitos e necessidades de usuário de um projeto de interface para um sistema de hospedagem. Com a coleta de dados finalizada, teve andamento o processo de design e foram identificados muitos problemas com regras de negócio, falta de funcionalidades e desrespeito às normas internas da empresa. Diante do exposto, o que pode ter dado errado?
>
> **A.**  
> O funcionário era novo e não tinha conhecimento suficiente para coletar as informações corretamente.
>
> **B.**  
> O tempo para o levantamento dos requisitos foi insuficiente.
>
> **C.**  
> A quantidade de usuários participantes foi muito grande.
>
> **D.**  
> Os usuários não sabiam dizer o que queriam da interface.
>
> **E.**  
> Os usuários não estavam disponíveis para participação na coleta de dados.
>> [!question]- Resposta
>>
>> **Resposta correta: A. O funcionário era novo e não tinha conhecimento suficiente para coletar as informações corretamente.**
>>
>> **Análise das alternativas:**
>>
>> **A. Certa.**  
>> O enunciado indica que, após a coleta de dados, surgiram problemas graves como falhas nas regras de negócio, ausência de funcionalidades e descumprimento de normas internas. Isso sugere que o levantamento de requisitos foi feito de forma inadequada, provavelmente por falta de experiência ou conhecimento suficiente do profissional responsável.
>>
>> **B. Errada.**  
>> O enunciado não informa que houve pouco tempo para o levantamento. Isso até poderia causar problemas, mas não é o indício principal apresentado na questão.
>>
>> **C. Errada.**  
>> Uma grande quantidade de usuários não é, por si só, um problema. Em muitos casos, ouvir mais usuários pode até enriquecer a coleta, desde que ela seja bem conduzida.
>>
>> **D. Errada.**  
>> Os usuários podem ter dificuldade em expressar exatamente o que desejam, mas o problema descrito envolve também regras de negócio e normas internas da empresa, que deveriam ser levantadas pela equipe de projeto.
>>
>> **E. Errada.**  
>> Se os usuários não estivessem disponíveis, a coleta provavelmente nem teria sido concluída adequadamente. O enunciado informa que a coleta foi finalizada, então esse não parece ser o problema principal.
>>
>> Neste caso, o problema foi colocar um funcionário novo, sem conhecimento do domínio e do artefato, para realizar o levantamento de requisitos junto ao usuário, pois é preciso conhecer bem o projeto e saber captar informações adequadamente para que a coleta de dados seja efetiva.


### 3
> [!question] Exercício 3
> Você e seus colegas de equipe de design estão se preparando para realizar o levantamento de requisitos e necessidades dos usuários de um novo artefato.
>
> Para este final de semana, foi programado que algumas poucas pessoas, diferentes dos usuários participantes de fato, viriam até a sua empresa, a fim de lerem com atenção todos os enunciados das questões utilizadas na coleta de dados, além de responderem aos questionários e às entrevistas, apontando, ainda, os problemas identificados. Qual é essa atividade?
>
> **A.**  
> Definição dos objetivos.
>
> **B.**  
> Planejamento.
>
> **C.**  
> Triangulação.
>
> **D.**  
> Coleta de dados ocasional.
>
> **E.**  
> Estudo piloto.
>> [!question]- Resposta
>>
>> **Resposta correta: E. Estudo piloto.**
>>
>> **Análise das alternativas:**
>>
>> **A. Errada.**  
>> **Definição dos objetivos** é a etapa em que se estabelece o que se pretende descobrir ou avaliar. No enunciado, a atividade já está em fase de teste dos instrumentos de coleta, não de definição de objetivos.
>>
>> **B. Errada.**  
>> **Planejamento** é uma fase mais ampla, que inclui organizar etapas, métodos e participantes. O caso descrito é mais específico: trata-se de testar previamente questionários e entrevistas antes da aplicação real.
>>
>> **C. Errada.**  
>> **Triangulação** é o uso de múltiplos métodos, fontes ou avaliadores para obter uma visão mais confiável dos dados. Não é isso que está acontecendo no enunciado.
>>
>> **D. Errada.**  
>> **Coleta de dados ocasional** não corresponde ao procedimento descrito. Aqui não se trata de uma coleta informal, mas de um teste prévio estruturado dos instrumentos.
>>
>> **E. Certa.**  
>> **Estudo piloto** é justamente a aplicação prévia de questionários, entrevistas ou outros instrumentos com um pequeno grupo, diferente dos participantes principais, para identificar falhas, ambiguidades e problemas antes da coleta definitiva.
>>
>> O **estudo piloto é feito antes da efetiva coleta de dados,** objetivando garantir que ele foi elaborado de modo adequado para a obtenção de todos os dados necessários para que o projeto do _design_ possa se fundamentar. No estudo piloto, pessoas que não vão participar da coleta real apontam problemas com os instrumentos que serão utilizados.

### 4
> [!question] Exercício 4
> Você e seus colegas designers estão desenvolvendo um projeto de interface para uma instituição bancária e escolheram um grupo pequeno de clientes para participarem da fase de coleta dados, informando os seguintes dados: nome, idade, sexo, agência, conta bancária, saldo em conta, salário atual e quantidade de transferências mensais.
>
> Vocês distribuíram os questionários para obter as informações dos participantes mas, para sua surpresa, muitos deles se negaram a preenchê-los. Qual foi o motivo?
>
> **A.**  
> O questionário é muito extenso.
>
> **B.**  
> O questionário apresenta perguntas de cunho pessoal e particular.
>
> **C.**  
> Os participantes preferiam responder a uma entrevista.
>
> **D.**  
> Clientes não podem participar de coletas de dados.
>
> **E.**  
> Os participantes deveriam ser os gestores do banco.
>> [!question]- Resposta
>>
>> **Resposta correta: B. O questionário apresenta perguntas de cunho pessoal e particular.**
>>
>> **Análise das alternativas:**
>>
>> **A. Errada.**  
>> O enunciado não informa que o problema era o tamanho do questionário. A recusa parece estar mais ligada ao tipo de informação solicitada do que à extensão.
>>
>> **B. Certa.**  
>> O questionário pede dados sensíveis e pessoais, como **conta bancária, saldo em conta e salário atual**. Esse tipo de informação é muito particular e pode gerar desconforto, insegurança e recusa por parte dos participantes.
>>
>> **C. Errada.**  
>> O enunciado não diz que os participantes preferiam entrevista. A principal pista dada é o conteúdo sensível das perguntas.
>>
>> **D. Errada.**  
>> Clientes podem, sim, participar de coletas de dados, especialmente quando são usuários ou público-alvo do sistema.
>>
>> **E. Errada.**  
>> Os gestores do banco podem ser stakeholders importantes, mas isso não significa que somente eles deveriam participar. O problema descrito está no teor das perguntas feitas aos clientes.
>>
>> Os _designers_ ou pesquisadores não podem incluir perguntas nos instrumentos de coleta que analisem aspectos da vida pessoal dos participantes. Nesse caso, por exemplo, não se pode solicitar que eles informem números de conta, saldos e salários.

### 5
> [!question] Exercício 5
> Você e seus colegas de equipe de design chamaram os gestores de uma cliente, que solicitou um artefato, para uma reunião de apresentação dos requisitos levantados. Durante o encontro, foram exibidos slides com problemas que os funcionários que participaram da coleta encontraram nos processos internos da empresa. O nome do funcionário apareceu junto ao problema identificado e, nesse momento, os gestores demonstraram descontentamento e intenção de punir os colaboradores. Qual princípio da pesquisa científica foi descumprido?
>
> **A.**  
> Princípio da não-maleficência.
>
> **B.**  
> Princípio da equidade.
>
> **C.**  
> Princípio da justiça.
>
> **D.**  
> Princípio da beneficência.
>
> **E.**  
> Princípio da autonomia.
>> [!question]- Resposta
>>
>> **Resposta correta: D. Princípio da beneficência.**
>>
>> **Análise das alternativas:**
>>
>> **A. Errada.**  
>> A **não-maleficência** está ligada à ideia de não causar dano, mas, nesta questão, o foco do gabarito esperado está na obrigação de a equipe agir para **minimizar ou eliminar riscos** aos participantes.
>>
>> **B. Errada.**  
>> **Equidade** se relaciona ao tratamento equilibrado dos participantes, o que não é o ponto central do problema apresentado.
>>
>> **C. Errada.**  
>> O princípio da **justiça** envolve tratamento justo e distribuição adequada de riscos e benefícios, mas não é o conceito mais diretamente associado ao caso descrito.
>>
>> **D. Certa.**  
>> O princípio da **beneficência** diz que a equipe de designers deve se preocupar em **minimizar ou eliminar totalmente qualquer risco de dano** que possa existir para os participantes. Neste caso, ao expor os nomes dos funcionários que encontraram problemas nos processos, esse princípio não foi seguido, pois a equipe não evitou que os colaboradores fossem prejudicados ou punidos.
>>
>> **E. Errada.**  
>> A **autonomia** está relacionada ao consentimento e à liberdade de decisão dos participantes, o que não é o aspecto principal violado no enunciado.
>>
>> O princípio da beneficência diz que a equipe de _designers_ deve se preocupar em minimizar ou eliminar totalmente qualquer risco de dano que possa existir para os participantes. Neste caso, expondo os nomes dos funcionários que encontraram problemas nos processos, tal princípio não foi seguido e a equipe não conseguiu evitar que os colaboradores fossem punidos.
